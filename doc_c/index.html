FatBeagle Library

FatBeagle is a wrapper than libhmsbeagle (see http://code.google.com/p/beagle-lib). (Ironically enough, FatBeagle is actually a very thin wrapper around libhmsbeagle; most of the hard work is done by libhmsbeagle).

The goals of FatBeagle are to:

  1. provide a slightly richer interface than libhmsbeagle. In particular, FatBeagle will calculate the eigensolution from an instantaneous rate matrix, and also provides functions that make it easier to apply some commonly-used forms of among-site rate variation.
  2. provide a C-language adaptor for the python-C-extension package, pytbeaglehon. FatBeagle is compiled from the same code-base that is the C-part of pytbeaglehon.

The documentation is currently very poor. Some function-level documentation is generated from doxygen.

Because the implementation of the libhmsbeagle API could be operating in what amounts to a distinct address space (e.g. if you are using a GPU resource through libhmsbeagle, the memory associated with partial likelihood arrays will be stored on the GPU, and not easily accessible by the CPU), you must initialize an environment "upfront." This requires to steps:

  1. A call to createLikelihoodCalcInstance which allocates the necessary memory and returns a integer handle;
  2. A call te getLikeCalculatorInstance to obtain a pointer to a LikeCalculatorInstance object that provides an interface to the environment for likelihood calculations
